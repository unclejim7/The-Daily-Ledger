# 📰 The Daily Ledger

> *A personalized daily broadsheet newspaper — generated by AI, published to the web.*

**Live site:** [unclejim7.github.io/The-Daily-Ledger](https://unclejim7.github.io/The-Daily-Ledger)

-----

## What Is This?

The Daily Ledger is a fully custom, AI-generated HTML newspaper published daily via GitHub Pages. Each edition is a richly formatted broadsheet covering markets, macroeconomics, Chicago sports, Chicagoland weather, real estate, health & wellness, current events, tax & accounting, and Federal Reserve news — curated and written fresh every day.

Every edition is generated on demand using a single command typed into Claude:

```
run tdl
```

Claude acts as writer and chief editor — searching the web across all coverage categories simultaneously, pulling live weather from the National Weather Service, composing articles, and assembling a complete single-file HTML newspaper in dark mode broadsheet style.

## File Structure

```
/
├── index.html              ← Today's edition (homepage)
├── archive.html            ← Full archive of all past editions
├── YYYY-MM-DD.html         ← Permanent dated copy of each edition
└── README.md
```

Each edition includes a **navigation bar** linking to the previous edition, the archive, and (once generated) the next edition — creating a fully browsable chain of issues.

-----

## How It Works

1. The `run tdl` command is issued to Claude in a dedicated Claude.ai project
1. Claude fetches the current date and time
1. Claude runs parallel web searches across all coverage categories
1. Claude fetches the Chicagoland weather forecast from the National Weather Service
1. Claude generates a complete HTML edition and writes 4 output files:
- `index.html` — today’s edition as the homepage
- `YYYY-MM-DD.html` — today’s permanent dated archive copy
- `archive.html` — updated archive page with the new edition on top
- Previous `YYYY-MM-DD.html` — patched with a live forward-navigation link to today
1. Files are downloaded and pushed to this repository manually
1. GitHub Pages serves the updated site within seconds

-----

## Updating the Site

After each `run tdl` run, download the 4 generated files from Claude and push them to this repo:

```bash
git add index.html archive.html YYYY-MM-DD.html PREV-YYYY-MM-DD.html
git commit -m "Daily Ledger: [date]"
git push
```

GitHub Pages picks up the changes automatically. The live site updates within ~30 seconds.

-----

## Archive

All past editions are permanently preserved as dated HTML files and linked from [archive.html](https://unclejim7.github.io/The-Daily-Ledger/archive.html). Editions are grouped by month and year, with the newest issue always shown at the top with a **Latest** badge.
